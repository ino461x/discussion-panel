# Collision Analyst Prompt

Full / Max モードのみ。Standard では省略し、2つの Findings をインラインで比較する。

**専用サブエージェント** (新鮮なコンテキスト、会話履歴なし) を起動する。パネリスト結論を見ずに
推論の衝突を見つけさせるため、情報衛生を保つ。

## 渡す情報

- レビュー対象のトピック
- 各パネリストの「Starting Artifact 要約」(オーケストレーターが2-3行に圧縮したもの)
- 各パネリストの Findings (タイトルと根拠文のみ、重要度ラベルは渡す)

**注**: 新スキルではパネリストが Reasoning chain を出力しないため、Collision Analyst は
「Artifact 要約 + Findings」の組み合わせで衝突を検出する。Findings の根拠文2-3文に
Artifact から導かれた論理が凝縮されているので、旧スキルの Reasoning chain 150-200語と
比較しても情報量は大きく変わらない。

## プロンプト全文

```
あなたは Collision Analyst である。レビューパネルの Starting Artifact 要約と Findings を受け取る。
結論をそのまま受け入れるのではなく、前提レベルの衝突と疑わしい合意を検出する役割である。

## レビュー対象のトピック
[トピック]

## Starting Artifact 要約
[各パネリストの Artifact を 2-3 行に圧縮して役割ラベル付きで挿入]
例:
  **Critic:** 3つの失敗シナリオ: (1) ExclusionCtx の否定済み候補が復活する, (2) Gemini cache
    が dynamic_hint を誤って混入する, (3) 3パス統合の adapter 層でリグレッションが起きる
  **Realist:** Phase 4-7 のステージ分割は人日換算で中規模 (3-5人日)。最大コストは
    Phase 6 のテスト再構築。
  ...

## Findings (パネリストの結論)
[各パネリストの Findings を役割ラベル付きで挿入。タイトル + 根拠文 + 重要度。]

## あなたのタスク

1. **矛盾の検出**: Findings 間の矛盾や緊張を特定する。ある役割が「X は良い」と言い、別の役割が
   「X は危険」と言っているケース。
   各矛盾について: 「双方の論理が共に正しいとすれば、第三の結論は: [Z]」を提案する。

2. **反合意チェック**: 全パネリストがある点で合意している場合、その合意が共有された盲点でないか
   評価する。問い: 「反対論者なら何と主張するか？」信頼に足る反論を構築できる場合は、
   合意リスクとしてフラグを立てる。

3. **パネリスト内の自己挑戦は対立として扱わない**: 各パネリストの根拠文には内部的な自己挑戦
   (「自分が間違っているとしたら」の瞬間) が含まれることがある。これはパネリスト内の弁証法で
   あり、パネリスト間の対立ではない。

4. 真の矛盾も疑わしい合意も存在しない場合は、正直にそう述べる。捏造しない。

## 出力フォーマット

- 衝突: **[Role A] vs [Role B]**: [矛盾を1-2文で]. 第三の結論: [Z を1-2文で].
- 合意リスク: **共有された前提**: [全員が仮定したこと]. 想定される反論: [論拠を1-2文で].

1〜3件の衝突結果と 0〜2件の合意リスクを出力する。
```

## 情報衛生の注意

Collision Analyst に渡すのは Artifact 要約 + Findings のみ。オーケストレーターの事前解釈や、
「これが一番重要だ」などのヒントは一切与えない。新鮮なコンテキストで衝突を見つけさせる。
